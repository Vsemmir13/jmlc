## 2. Проблема

При работе с большими некураторскими коллекциями изображений (web crawl и пользовательский контент) полная разметка через тяжёлые VLM или ручную аннотацию становится **слишком дорогой по времени и вычислениям**.

Нужен **лёгкий фильтр первого уровня** — multilabel-классификатор, который:

1. выделяет релевантные срезы данных для downstream vision-задач;
2. отсекает заведомо нерелевантный / мусорный контент;
3. масштабируется на batch-inference и деплой (ONNX).

## 3. Подход (публичная версия)

### 3.1. Подготовка данных (обобщённо)

В prod-версии использовался многоэтапный pipeline: embedding-based pre-filtering, слабая разметка ансамблем VLM, агрегация меток. **Детали этого этапа в репозиторий не включены** из-за NDA.

Для демонстрации воспроизводимости используется синтетический demo-набор `data/demo/` с 4 классами (`class_0` … `class_3`).

### 3.2. Модель

- Backbone: **EVA02-Base/14@448** (публичная архитектура, совместимая с timm-checkpoints).
- Head: FC + GELU, multilabel sigmoid.
- Loss: **DB loss** (class-balanced focal-style loss для long-tail распределения меток).

Альтернативно в коде поддерживаются ResNet, EfficientNet и другие backbones.

### 3.3. Обучение и оценка

- Конфигурация через YAML.
- Mixed precision (bf16) на GPU.
- Distributed training (DDP) через `torchrun`.
- Метрики: accuracy, F1, precision, recall, per-class F1, ROC-AUC, PR-AUC.
- Checkpointing лучших моделей.

### 3.4. Inference и деплой

- Batch inference через `src/inferencer.py` (строки с base64 или file paths).
- Экспорт в ONNX для production-serving.

Prod map-reduce job на внутренней платформе **не включён**; публичный API inference сохранён в переносимом виде.

## 6. Мой вклад

Весь код в этом репозитории **реализован мной лично** (участником конкурса), включая:

- постановку задачи multilabel-фильтрации изображений и выбор метрик (PR-AUC, macro/micro F1);
- проектирование и реализацию пайплайна обучения: `main.py`, `trainer.py`, `dataset.py`, `config.py`;
- реализацию модели и поддержку нескольких backbone (`model.py`, локальный `eva.py` с совместимостью timm-checkpoints);
- реализацию **DB loss** для long-tail multilabel (`losses.py`) и интеграцию в training loop;
- модуль оценки с multilabel-метриками и поддержкой DDP (`evaluator.py`);
- batch-inference API (`inferencer.py`) и экспорт в ONNX (`export_onnx.py`);
- YAML-конфигурацию экспериментов, checkpointing, опциональное логирование в Wandb;
- подготовку публичной версии репозитория для JMLC: demo-данные, smoke test (`scripts/smoke_demo.py`), документация и NDA-sanitization.

Prod-интеграции (autolabel, YT map-reduce, закрытые данные) в публикацию не входят по NDA; **ML-ядро в репозитории — моя полная самостоятельная реализация.**
